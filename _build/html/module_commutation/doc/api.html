Commutation API¶

Client API¶

void set_commutation_sinusoidal(chanend c_commutation, int input_voltage)¶

Sets input voltage for commutation loop.

Parameters:
  • c_commutation – A chanend connected to commutation server
  • input_voltage – Motor input voltage (range: -13739 to 13739)

Server API¶

void init_commutation_param(commutation_par &commutation_params, hall_par &hall_params, int nominal_speed)¶

Initialize commutation parameters.

Parameters:
  • commutation_params – struct defines the commutation angle parameters
  • hall_params – struct defines the pole-pairs and gear ratio
  • nominal_speed – is the rated speed for the motor given on specs sheet
void commutation_sinusoidal(chanend c_hall, chanend c_qei, chanend c_signal, chanend c_watchdog, chanend c_commutation_p1, chanend c_commutation_p2, chanend c_commutation_p3, chanend c_pwm_ctrl, out port p_ifm_esf_rstn_pwml_pwmh, port p_ifm_coastn, port p_ifm_ff1, port p_ifm_ff2, hall_par &hall_params, qei_par &qei_params, commutation_par &commutation_params)¶

Sinusoidal based Commutation Loop.

Parameters:
  • c_hall – A chanend connected to the hall server
  • c_qei – A chanend connected to the qei server (QEI)
  • c_signal – A chanend for signaling after initialization of commutation loop
  • c_watchdog – A chanend connected to the watchdog
  • c_commutation_p1 – channel for receiving motor voltage input value - priority 1 (highest) 1 ... (lowest) 3
  • c_commutation_p2 – channel for receiving motor voltage input value - priority 2
  • c_commutation_p3 – channel for receiving motor voltage input value - priority 3
  • c_pwm_ctrl – channel to set PWM level output to motor phases
  • p_ifm_esf_rstn_pwml_pwmh – port to configure motor FET driver
  • p_ifm_coastn – port to enable motor FET driver
  • p_ifm_ff1 –
  • p_ifm_ff2 –
  • hall_params – struct defines the pole-pairs and gear ratio
  • qei_params – the struct defines sensor type and resolution parameters for QEI
  • commutation_params – struct defines the commutation angle parameters
